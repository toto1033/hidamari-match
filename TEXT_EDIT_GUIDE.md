# ひだまりマッチ テキスト編集ガイド

## 基本の流れ

```
① VS Codeでファイルを開く
② テキストを書き換える
③ Ctrl + S で保存
④ ターミナルで git add . → git commit → git push
⑤ Vercelに自動反映（数分）
```

---

## どこを開けばいい？早見表

### サイト上の場所 → 開くファイル

| 変えたい場所 | 開くファイル | 場所 |
|---|---|---|
| キャッチコピー「お子さまに〜」 | `src/app/page.tsx` | 上から30行目あたり |
| サブコピー「全国22,000以上〜」 | `src/app/page.tsx` | キャッチコピーの下 |
| ナビメニュー（施設を探す・使い方・施設掲載） | `src/components/Header.tsx` | 上から40行目あたり |
| 「無料で探す」ボタンの文字 | `src/components/Header.tsx` | ナビの右端 |
| タグ（送迎あり・個別支援・空きありなど） | `src/app/page.tsx` | 検索バーの下 |
| 統計数値（22,000+・47・3,400+） | `src/app/page.tsx` | タグの下のティールバー |
| 3ステップの説明文 | `src/app/page.tsx` | ページ中段 |
| フッターのリンク・コピーライト | `src/app/page.tsx` | 一番下 |
| 施設名・キャッチコピー・住所など | `src/data/facilities.ts` | ダミーデータ |

## VS Codeでの開き方

### 方法A：左のファイルツリーから開く

```
左のパネルを見る
  └ src フォルダをクリックして展開
      └ app → page.tsx をクリック
      └ components → Header.tsx をクリック
      └ data → facilities.ts をクリック
```

### 方法B：Ctrl + P で検索して開く

```
Ctrl + P を押す
→ ファイル名を入力（例：page.tsx）
→ Enterで開く
```

## テキストの探し方

### Ctrl + F で検索

ファイルを開いた状態で `Ctrl + F` を押すと検索バーが出ます。

```
例：「22,000」と入力すると統計数値の場所にジャンプ
例：「施設を探す」と入力するとナビの場所にジャンプ
```

## 実際の書き換え例

### 例1：キャッチコピーを変える

`src/app/page.tsx` を開いて `Ctrl + F` で「お子さまに」を検索：

```tsx
// 変更前
お子さまにぴったりの施設がきっと見つかる

// 変更後（自由に書き換える）
お子さまにぴったりの放課後デイサービスが見つかる
```

### 例2：統計数値を変える

`src/app/page.tsx` を開いて `Ctrl + F` で「22,000」を検索：

```tsx
// 変更前
22,000+

// 変更後
1,200+
```

### 例3：施設情報を変える

`src/data/facilities.ts` を開くと以下のような構造が並んでいる：

```typescript
{
  id: "1",
  name: "ひだまりキッズ 渋谷教室",      ← 施設名
  catchcopy: "一人ひとりの「できた！」を大切に",  ← キャッチコピー
  address: "東京都渋谷区桜丘町1-2-3",   ← 住所
  ...
}
```

## 保存してpushする手順

### ① ファイルを保存

```
Ctrl + S
```

タブのファイル名から「●」が消えたら保存完了。

### ② ターミナルを開く

```
Ctrl + ` （バッククォート）
```

### ③ GitHubにpush

```bash
git add .
git commit -m "テキストを修正"
git push
```

### ④ Vercelで確認

数分後に `hidamari-match.vercel.app` を開くと反映されている。

```

```


## よくあるミス

| ミス | 対処法 |
|---|---|
| 保存し忘れ | Ctrl + S を必ず押す |
| pushし忘れ | `git push` まで実行する |
| 反映されない | 数分待つ・Vercelのダッシュボードを確認 |
| エラーが出た | `git status` で状態確認・Claude Codeに相談 |
